Setup

Everything you need to go from a fresh laptop to a running PICARD frontend connected to the production backend.

Prerequisites

Before you can run anything, you'll need:

  • An approved WVU SSH gateway account (your <username>@ssh.wvu.edu)
  • An account on the PICARD server (Dr. Devine or a team member can create one)
  • Membership in the ThePICARDProject GitHub organization
  • npm installed locally (for the frontend)
  • Chrome (the frontend was designed against Chrome — Firefox tends to throw CORS errors)

SSH Setup

  1. Request gateway access. Submit the WVU service desk form. Use 157.182.194.132 as the destination IP.
  2. Verify gateway access. 
    ssh <your-username>@ssh.wvu.edu
  3. Get a PICARD server account. Have a team member create one for you.
  4. Hop from gateway to PICARD. 
    ssh <your-username>@picard.csee.wvu.edu

If you're logged in, you're set up. Move on to SSH config (recommended) or jump straight to Booting Up.

SSH Config (Optional but Recommended)

Configure an identity-based jump so you can ssh picard in one shot with the tunnel preconfigured.

  1. Open (or create) ~/.ssh/config.
  2. Add the following: 
    Host gateway
    HostName ssh.wvu.edu
    Port 22
    User <your school username>
    IdentityFile ~/.ssh/id_rsa
    LocalForward 5080 localhost:5080

Host picard
    HostName picard.csee.wvu.edu
    User <your school username>
    Port 22
    IdentityFile ~/.ssh/id_rsa
    ProxyJump gateway
    LocalForward 5080 localhost:5080
  3. If you don't have an id_rsa in ~/.ssh/, generate one: ssh-keygen -t rsa.
  4. Copy your key: ssh-copy-id picard (you'll be prompted for passwords a few times).
  5. Now ssh picard drops you straight in with port 5080 forwarded.

Booting Up

Get the frontend running locally and connected to the backend.

  1. Clone 481-FrontEnd to your local machine.
  2. Install dependencies: npm install (and npm install vite if needed).
  3. From the picard folder (typically /481-FrontEnd/picard), start the dev server: 
    npm run dev
    Press o + enter to open the page. It will load on localhost:5173 but will not work until the backend tunnel is active.
  4. In another terminal, SSH into the WVU gateway with port forwarding: 
    ssh -L 5080:127.0.0.1:5080 <username>@ssh.wvu.edu
    Heads up Use 127.0.0.1 rather than localhost — some systems don't resolve localhost consistently. If the command errors, try it again (sometimes it takes 2–3 attempts).
  5. From the gateway, SSH to the PICARD server with the same forward: 
    ssh -L 5080:127.0.0.1:5080 backend@157.182.194.132
    The current group username is backend. This may change in later semesters.
  6. Verify you're in /home/backend: 
    pwd
# if not:
cd /home/backend
  7. Check whether the backend is already running: 
    lsof -i :5080
# or
ps aux | grep API-backend
    If you see output, you're good. If not, start it manually: 
    cd /home/backend/API-backend
dotnet run
    Warning The dotnet instance dies if the terminal closes. Keep the session open while you're working.

Using the Site

  1. The frontend serves on localhost:5173.
  2. Click the login button. You'll be redirected to localhost:5080 (the tunneled backend).
  3. Sign in with a Google account ending in @mix.wvu.edu. Login may flake (whitepage, 404, freeze) — back out and retry; only treat it as broken after ~4 attempts.
  4. You'll land on the main dashboard.
Tip Stale cookies from previous sessions can produce odd behavior. If something looks off, clear cookies for localhost.

Running Experiments

  1. Click NEW EXPERIMENT.
  2. Click + Add New Algorithm. Use a preset from the algorithms repo while you're learning the system.
  3. Upload a CSV as your dataset.
  4. Pick your algorithm, configure parameters, name the experiment, and submit.
Reality check Algorithms that work locally may still fail on the cluster — and you won't find out until the swarm finishes spinning up (~3 minutes). Budget about 5 minutes per "is my submission valid" cycle and remember that the error could be anywhere in the upload chain.
